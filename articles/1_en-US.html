1 Raw Text Formatting

The language uses 5 Raw Text Formatting styles.

1.1 "Title" Formatting

All lines are trimmed, empty lines are removed, leftover lines are joined with a space

1.2 "Integer" Formatting

Same as "Title" Formatting but the result is parsed to a integer.

1.3 "Paragraph" Formatting

All lines are trimmed, empty lines are removed, leftover lines are joined with a line separator

1.4 "Code" Formatting

Blank lines are removed until a non blank line is found, leftover lines are joined with a line separator, the resulting string is then stripped from trailing white spaces and indentation.

1.5 "List" Formatting

Text is separated by a "," (Comma) (a comma can be escaped by using two of them ",,") then all separated text is formatted using "Title" Formatting and any empty separated text is removed.

2 Text Blocks

A Text Block starts with:

  • [blockname<attribute>]

And only ends with the exact sequence:

  • [;<;>]

There are no escape characters, "[]<>;" is not allowed.

The ending sequence can be escaped by placing a ";" after: [;<;>];

No text except white spaces are allowed outside a block.

A Block cannot contain a block.

2.1 "blockname"

"blockname" is formatted using "Title" formatting, the resulting block name is then converted to lower case.

Any white space inside the "blockname" is not allowed

2.2 "attribute"

"attribute" is fomatted as "blockname", but white spaces inside the attribute are allowed and the result is not converted to lower case.

3 Article

A Article is composed out of meta blocks and body blocks that is converted to HTML, attributes are used as the language of the block, a block without a language should always be inserted if it is a body block or used as a fallback if it is a meta block.

A article is only valid if it defines a "id" and "languages" block, the "languages" block must not be empty.

A article should always have a "id" and a "languages" block as the first and second block.

"id" and "languages" can not have a defined language.

4 Meta Blocks

Those are blocks that only appear at the start of the article.

They are not allowed to be declared after body blocks have started and can only be redeclared if the language is different.

4.1 id

Defines the id of the article, uses "Integer" formatting.

4.2 languages

Defines the languages of the article, uses "List" formatting.

4.3 license

Defines the license of the article, uses "Code" formatting.

4.4 title

Defines the title of the article, uses "Title" formatting.

4.5 description

Defines the description of the article, uses "Title" formatting.

4.6 date

Defines the date of the article, uses "Title" formatting

4.7 footer-return

Defines the return to main articles link text, uses "Title" formatting

4.8 footer-notice

Defines the copyright notice at the of the article, uses "Title" formatting

5 Body Blocks

All body blocks must have a preceding "section" or "subsection" block except "section" and "subsection".

5.1 section

Defines a section, uses "Title" formatting.

5.2 subsection

Defines a subsection, uses "Title" formatting, can only be declared if a "section" was already declared.

5.3 text

Defines a paragraph, uses "Paragraph" formatting.

5.4 fine

Defines a "fine" paragraph, uses "Paragraph" formatting.

5.5 warning

Defines a "warning" paragraph, uses "Paragraph" formatting.

5.6 severe

Defines a "severe" paragraph, uses "Paragraph" formatting.

5.7 code

Defines a code block, uses "Code" formatting.

5.8 image

Defines a image, uses "Title" formatting.

5.9 olist

Defines a ordered list, uses "List" formatting.

5.10 ulist

Defines a unordered list, uses "List" formatting.

5.11 html

Defines raw html without any processing, uses "Code" formatting.

6 Font Formatting

Text can be further processed by using font formatting with it.

Font formatting can be escaped by using a \

  • ///italic/// -> italic

  • ***bold*** -> bold

  • ```code``` -> code

  • ~~~crossed~~~ -> crossed

  • ___underline___ -> underline

  • [[[../resources/bird.jpg:::Bird]]] -> Bird

  2. return text
  3. .lines()
  4. .map((s) -> s.trim())
  5. .filter((s) -> !s.isEmpty())
  6. .collect(Collectors.joining(" "));
  7. }
  9. public static String getParagraphFormatted(String text) {
  10. return text
  11. .lines()
  12. .map((s) -> s.trim())
  13. .filter((s) -> !s.isEmpty())
  14. .collect(Collectors.joining("\n"));
  15. }
  17. public static String getCodeFormatted(String text) {
  18. return text
  19. .lines()
  20. .filter(new Predicate<String>() {
  21. boolean startFound = false;
  23. @Override
  24. public boolean test(String t) {
  25. if (t.isBlank() && !this.startFound) {
  26. return false;
  27. }
  28. this.startFound = true;
  29. return true;
  30. }
  31. })
  32. .collect(Collectors.joining("\n"))
  33. .stripTrailing()
  34. .stripIndent();
  35. }
