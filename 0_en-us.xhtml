Texture Format Size Per PixelNotes
RGB32F 16 bytes Overkill for colors, due to padding has the same size as RGBA32F
RGB16F 8 bytes Good for colors, due to padding has the same size as RGBA16F
RGBA8 RGBE 4 bytes Has linear interpolation problems due to exponential values
RGBA8 RGBM 4 bytes Has precision or hue shifting problems due to fixed precision
DXT5 RGBE 1 byte Same as RGBA8 RGBE but compressed in DXT5
DXT5 RGBM 1 byte Same as RGBA8 RGBM but compressed in DXT5
BC6H 1 byte Modern texture compression for HDR Textures, may not work in older hardware tools or libraries for compression are more scarce, I recommend using NVIDIA Texture Tools
DXT1 sRGB * Intensity0.5 bytes Highly agressive compression, may have precision problems.
Tipo de Anti-Aliasing Descrição
MSAA (Multi-Sample Anti-Aliasing) Reduz o serrilhado suavizando bordas de objetos, amostrando várias posições dentro de cada pixel.
FXAA (Fast Approximate Anti-Aliasing) Aplica um filtro pós-processamento para suavizar bordas, menos intensivo em termos de recursos.
TXAA (Temporal Anti-Aliasing) Combina MSAA com informações temporais para reduzir o serrilhado e preservar detalhes em movimento.
SMAA (Subpixel Morphological Anti-Aliasing)Um método de AA baseado em shaders que pode oferecer resultados de alta qualidade com impacto moderado nos recursos.
CSAA (Coverage Sample Anti-Aliasing) Uma versão melhorada do MSAA pela NVIDIA, utilizando amostragem de cobertura para melhorar a qualidade de imagem.
